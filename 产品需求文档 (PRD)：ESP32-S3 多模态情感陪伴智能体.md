产品需求文档 (PRD)
所属系列：从0到1打造“情感陪伴智能体”系列的综合落地指南
文档状态：已确认（V4.1 极简几何 UI 演进版）
项目代号：xiapiya

1. 项目概述
1.1 核心目标
打造一款具备“听、说、看、表达”四维能力的桌面具身智能终端。基于 ESP32-S3 的边缘算力与云端服务器的强大调度能力，结合通义千问多模态大模型（Qwen-VL + Omni），实现多模态情感交互。本版本（V4.1）在视觉表现上采用“极简几何风（Minimalist UI）”，剥离复杂的拟真表情，将系统资源最大化倾斜于音频流传输与网络调度，确保工业级的流畅度与稳定性。

1.2 黄金硬件底座与底层规范
开发框架： 纯 ESP-IDF (v5.x) - 严禁使用 Arduino 框架封装，确保内存管理与多线程调度的工业级稳定性。

主控大脑： ESP32-S3-N16R8（双核，16MB Flash + 8MB PSRAM，完美支持 LVGL 与大容量图像缓存）。

音频系统： INMP441 (I2S 录音) + MAX98357A (I2S 功放播放) + 3W 扬声器。

视觉系统： OV2640 摄像头模组。

交互系统： SPI 接口 TFT 屏幕。

1.3 云端基础设施
服务器环境： 标准 Linux 云服务器（使用 1Panel 面板进行可视化的容器化管理）。

模型网关： 部署 NewAPI 统一管理 Qwen 等大模型的 API Key 与请求并发，实现接口的高可用与账单收敛。

2. 极简 UI 交互规范 (TFT + LVGL)
使用 SquareLine Studio 进行静态布局导出，并在 C 代码中手动控制状态转移与颜色变化。取消所有滚动字幕与拟真表情。

2.1 顶部状态栏 (Status Bar)
元素： 仅保留 WiFi 与 MQTT 极简图标。

逻辑： 状态改变时仅变色，不弹窗。已连接显示为绿色（或主题色），断开显示为灰色。

2.2 中心显示区状态机 (Core States)
中心区域根据当前业务生命周期切换纯几何图形，提供直观的设备状态感知：

待机态 (Idle)： 中心区域完全留白（或仅保留极暗的屏幕背光），防止烧屏并降低功耗。

倾听态 (Listening)： VAD 唤醒或手动触发录音时，中心显示静态的三个点 ...，无任何动画，保持克制。

思考态 (Thinking)： 音频已上传，等待大模型处理网络延迟时，中心显示闪烁的三个点 ...（高低透明度交替），掩盖网络耗时。

说话态 (Speaking)： 接收并播放云端下发的 TTS 音频流时，中心显示一个“空心圆”，并配合音频流执行平滑的缩放/透明度呼吸动画（由 LVGL lv_anim_t 实现）。

视觉抓拍态 (Capturing)： 执行抓拍动作瞬间，屏幕闪烁一次白光（模拟快门），随后切回原状态。

3. 核心业务流程与通信解耦
3.1 公网双通道安全传输
设备接入公网后，采用“重数据走 HTTPS，轻数据走 MQTTS”的策略保障链路安全与实时性：

MQTTS (EMQX Docker)： 开启 TLS/SSL 加密。负责全双工的实时控制流（UI 状态切换指令）、设备状态流、以及 Base64/Raw 音频数据的下发。

HTTPS (FastAPI)： ESP32 唤醒 OV2640 抓拍 JPEG 照片后，通过 HTTPS POST 直接向云端 FastAPI 上传图片。

3.2 交互触发逻辑
日常陪伴： VAD 唤醒 -> 屏幕切至 ... 录音 -> 上传音频 -> 屏幕闪烁 ... 等待 -> 屏幕切至呼吸圆，播放音频 -> 回归待机。

视觉触发： 用户下达指令或移动端下发指令 -> 执行抓拍动作 -> HTTPS 上传并附带上下文 -> 多模态推理。

4. 云端大脑与多端协同
4.1 Python / FastAPI 逻辑中枢 (Docker 部署)
路由分发： 接收硬件侧的 HTTPS 图像数据。

网关调度： 通过本地的 NewAPI 聚合接口请求 Qwen-VL/Omni，剥离大厂商 SDK 依赖。

指令编排： 处理推理结果后，通过 MQTT 先下发 UI 状态切换指令（切换至说话态），再下发音频流数据。

4.2 Web 监控端 (上帝视角)
挂载在 Nginx 下的轻量级 HTML/JS 面板。通过 WSS 连接 EMQX，实时监听并打印设备全链路日志、抓拍留存记录以及 API 耗时监控。

4.3 Android 移动端 (移动数字分身 - V4.2极简解耦版)
**核心定位：** 作为智能体的移动文本客户端，完全解耦硬件控制，专注纯文本对话。

**技术栈：** Kotlin + Retrofit + OkHttp-SSE + Room + MVVM

**核心功能：**
- **文本对话：** 流式渲染（SSE），聊天气泡界面
- **跨端记忆同步：** 通过硬编码Session ID共享云端上下文（可访问ESP32的语音对话历史）
- **本地持久化：** Room数据库存储聊天历史，支持离线查看
- **免鉴权设计：** 硬编码`session_id: xiapiya_master_01`，快速迭代

**设计原则：**
- 极致减法：剔除远程拍照、静音开关等硬件控制
- 完全解耦：不依赖ESP32在线状态，独立对话模式
- 统一意识：通过云端上下文池实现跨端记忆连贯性

5. 开发路线图 (敏捷迭代版)
Phase 1: 云端基建与硬件基础 (第 1 周)

利用 1Panel 部署 EMQX、NewAPI 与 MySQL 容器，跑通 MQTT 连通性测试。

焊接主板，基于 ESP-IDF 跑通 I2S MIC、功放及 TFT 屏幕的底层驱动。

Phase 2: 极简 UI 与通信闭环 (第 2-3 周)

将 SquareLine Studio 导出的代码集成至项目，手写 lv_anim 实现呼吸圆逻辑。

编写 FastAPI 基础路由，跑通“录音-上传-思考-下发-播放”的单模态完整状态机链路。

Phase 3: 视觉接入与网页中枢 (第 4-5 周)

联调 OV2640，实现 HTTPS 图片直传，后端接入 Qwen-VL 实现多模态理解。

部署 Web 监控端，完成全链路可视化。

Phase 4: Android 生态与优化交付 (第 6-8 周)

完成 Android 文本客户端（V4.2极简版）：
- Kotlin + Retrofit + SSE流式渲染
- MVVM架构（ViewModel + Repository + Room）
- 硬编码Session ID，跨端上下文验证
- 聊天界面与本地持久化

部署 Web 监控端（WSS实时日志可视化）。

进行弱网环境下的状态机异常测试（防卡死处理）与内存泄漏排查（重点监控 PSRAM 图像缓存与 LVGL 内存池）。
