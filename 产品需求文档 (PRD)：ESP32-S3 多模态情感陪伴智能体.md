## 1. 项目概述

### 1.1 核心目标

打造一款具备“听、说、看、表达”四维能力的桌面具身智能终端。基于 ESP32-S3 的边缘算力与**云端服务器**的强大调度能力，结合通义千问多模态大模型（Qwen-VL + Omni），不仅能听懂用户的情绪，还能在特定场景下“看见”用户的状态。通过 TFT 屏幕呈现丰富的表情与 UI 交互，辅以多端（Web/App）协同，提供极具温度的情感陪伴。

### 1.2 黄金硬件底座与底层规范

- **开发框架：** **纯 ESP-IDF (v5.x) - 严禁使用 Arduino 框架封装**，确保内存管理与多线程调度的工业级稳定性。
    
- **主控大脑：** ESP32-S3-N16R8（双核，16MB Flash + 8MB PSRAM，内存自由，完美支持 LVGL 与大容量图像缓存）。
    
- **音频系统：** INMP441 (I2S 麦克风录音) + MAX98357A (I2S 功放播放) + 3W 扬声器。
    
- **视觉系统：** OV2640 摄像头模组。
    
- **交互系统：** SPI 接口 TFT 屏幕。
    

### 1.3 云端基础设施 (新增)

- **服务器环境：** 标准 Linux 云服务器（推荐使用 **1Panel** 面板进行可视化的容器化管理，降低后期运维心智负担）。
    
- **模型网关：** 部署 **NewAPI**（或类似网关服务）统一管理 Qwen 等大模型的 API Key 与请求并发，实现接口的高可用与账单收敛。
    

## 2. 核心业务流程与架构设计

### 2.1 交互触发逻辑 (按需注视)

- **日常语音陪伴：** 默认通过 VAD 触发音频流录制与对话。
    
- **视觉多模态触发：** 当用户下达特定语音指令（如：“看看我这件衣服”），或通过 App/Web 下发“请求视觉”指令时，ESP32 唤醒 OV2640 抓拍当前帧画面。
    

### 2.2 通信链路解耦 (公网双通道安全传输)

设备接入公网后，必须保障链路安全，采用“重数据走 HTTPS，轻数据走 MQTTS”的策略：

- **MQTTS (EMQX Docker 部署)：** 开启 TLS/SSL 加密。负责全双工的实时控制流、设备状态流、以及 Base64 音频数据的下发与上传。
    
- **HTTPS (FastAPI)：** ESP32 抓拍 JPEG 照片后，通过 HTTPS POST 直接向云端 FastAPI 上传图片，确保隐私数据在公网传输的安全性。
    

### 2.3 延迟补偿与 TFT 屏幕 UI (LVGL)

引入 LVGL 图形库实现现代化 UI，包含：表情区、动态字幕区、系统状态栏。全生命周期状态机映射需配合公网网络波动进行优化：

- **待机：** 播放“眨眼/呼吸”微动效，状态栏显示 WiFi 与 MQTTS 连接图标。
    
- **拍照中：** 显示“快门闪光”动效。
    
- **云端交互中 (掩盖网络与推理延迟)：** 云端通过 MQTT 实时下发状态节点，屏幕同步显示“上行同步中...”、“小脑瓜狂转中...”。
    
- **回复中：** 根据大模型返回的情感标签，屏幕切换对应主表情，同步滚动显示字幕，同时播放音频。
    

## 3. 云端大脑与多端协同 (服务器侧重点)

### 3.1 Python / FastAPI 逻辑中枢 (Docker 部署)

- **路由分发：** 接收音频（Webhook）与图像（HTTPS POST）请求。
    
- **网关调度：** 后端不直接请求厂商 API，而是统一将请求打向服务器内部部署的 **NewAPI** 服务，由其分发给 Qwen-VL/Omni，提高密钥安全性。
    
- **TTS 与转码：** 合成指定音色，转换为 32kbps MP3。
    
- **指令编排下发：** 先向 `emqx/esp32/ui` 下发包含文本和情绪的 JSON；再向 `emqx/esp32/playaudio` 下发音频流。
    

### 3.2 Web 监控端 (上帝视角)

- 提供轻量级 HTML/JS 面板，挂载在服务器的 Nginx 服务下。通过 WSS (WebSocket Secure) 连接 EMQX，实时打印设备的所有动作流、图像抓拍记录、耗时统计。
    

### 3.3 Android 移动端 (随身伴侣)

- 采用 Kotlin + HiveMQ 搭建的聊天气泡流界面。
    
- 包含远程语音对话、历史记录本地化存储（Room）。
    
- 提供“远程拍照”触发按钮与“静音 ESP32”硬件控制开关。
    

## 4. 里程碑与开发路线图 (敏捷迭代版)

- **Phase 1: 云端基建与硬件点亮 (第 1 周)**
    
    - **云端：** 配置服务器防火墙，利用 1Panel 部署 EMQX、NewAPI 与 MySQL 容器，跑通公网 MQTT 连通性测试。
        
    - **设备：** 焊接/拼装 ESP32-S3 + 外设，基于 ESP-IDF v5.x 跑通外设的初始化驱动。
        
- **Phase 2: 通信闭环与基础 UI (第 2-3 周)**
    
    - **设备：** 移植 LVGL，构建基础状态机动画；跑通 MQTTS 文本收发与 HTTPS 图片上传。
        
    - **云端：** 编写 FastAPI 基础路由，跑通单模态（纯语音）对话闭环。
        
- **Phase 3: 视觉大模型与网页中枢 (第 4-5 周)**
    
    - **系统：** 联调 OV2640 摄像头，后端接入 Qwen-VL 实现视觉语义提取。
        
    - **前端：** 部署 Web 监控端，实现链路日志与抓拍画面的实时可视化。
        
- **Phase 4: Android 生态与项目交付 (第 6-8 周)**
    
    - 完成 Android App 研发（Kotlin），接入 MQTT 实现多端协同。
        
    - 进行弱网环境下的延迟测试与异常重连机制优化，完成商业级系统闭环。
